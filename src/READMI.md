## Задание 1: Класс Дробь с кэшированием

### Формулировка задачи
Создать класс `Fraction`, представляющий математическую дробь с целыми числителем и знаменателем. Класс должен реализовать кэширование вещественного значения дроби.

### Требования
- Числитель и знаменатель - целые числа
- Знаменатель не может быть равен нулю
- Автоматическая нормализация знаков (знаменатель всегда положительный)
- Кэширование вычисленного вещественного значения
- Методы для установки числителя и знаменателя
- Переопределение методов `equals()`, `hashCode()`, `toString()`

### Алгоритм решения
1. Создание полей класса: числитель, знаменатель, кэшированное значение
2. Конструктор с проверкой знаменателя и нормализацией знаков
3. Метод `getRealValue()` с проверкой кэша
4. Методы `setNumerator()` и `setDenominator()` со сбросом кэша
5. Переопределение стандартных методов

## Задание 2: Класс Кот и интерфейс Мяуканье

### Формулировка задачи
Создать систему для работы с мяукающими объектами с подсчётом количества мяуканий.

### Требования
- Интерфейс `Meowable` с методом `meow()`
- Класс `Cat` с именем и счётчиком мяуканий
- Метод для многократного вызова мяуканья
- Возможность получения количества мяуканий

### Алгоритм решения
1. Создание интерфейса `Meowable`
2. Реализация класса `Cat` с полями: имя и счётчик
3. Реализация метода `meow()` с увеличением счётчика
4. Метод `meowsCare()` для вызова мяуканья 5 раз

## Задание 3: Обработка списков

### Формулировка задачи
Создать новый список из элементов, которые присутствуют только в одном из двух исходных списков.

### Требования
- Элементы, которые есть в L1, но нет в L2
- Элементы, которые есть в L2, но нет в L1
- Каждый элемент только один раз в результате
- Не включать элементы, присутствующие в обоих списках

### Алгоритм решения
1. Создание результирующего списка
2. Первый проход по L1: проверка отсутствия в L2
3. Второй проход по L2: проверка отсутствия в L1
4. Проверка уникальности в результате
5. Возврат результата

## Задание 4: Обработка данных абитуриентов

### Формулировка задачи
Прочитать данные из файла об абитуриентах и найти тех, кто не допущен к экзаменам.

### Требования
- Чтение данных из текстового файла
- Формат: количество абитуриентов, затем строки "Фамилия Имя балл1 балл2"
- Допуск: ≥30 баллов по каждому предмету
- Вывод списка не прошедших
- Использование Map для хранения данных

### Алгоритм решения
1. Чтение файла с помощью BufferedReader
2. Обработка каждой записи: разделение строки, извлечение данных
3. Проверка условий допуска (оба балла ≥30)
4. Хранение данных в Map: ключ - имя, значение - баллы
5. Формирование списка не прошедших

## Задание 5: Поиск звонких согласных

### Формулировка задачи
Найти в тексте звонкие согласные буквы, которые встречаются более чем в одном слове.

### Требования
- Чтение русского текста из файла
- Определение звонких согласных (б, в, г, д, ж, з, й, л, м, н, р)
- Подсчёт в скольких уникальных словах встречается каждая согласная
- Вывод в алфавитном порядке
- Учитывать только буквы, встречающиеся более чем в одном слове

### Алгоритм решения
1. Определение множества звонких согласных
2. Чтение и очистка текста
3. Разделение на слова, игнорирование пустых слов
4. Подсчёт согласных по словам с использованием Set для уникальности
5. Фильтрация (счётчик > 1) и сортировка по алфавиту

## Задание 6: Обратный порядок очереди

### Формулировка задачи
Вывести элементы непустой очереди в обратном порядке.

### Требования
- Использование стандартной структуры Queue
- Вывод без изменения исходной очереди
- Использование стека для реверса порядка

### Алгоритм решения
1. Проверка очереди на пустоту
2. Создание стека для временного хранения
3. Создание копии очереди (чтобы не изменять оригинал)
4. Перенос элементов из очереди в стек
5. Вывод элементов из стека (обратный порядок)

## Задание 7: Обработка точек стримом

### Формулировка задачи
Обработать список точек и создать ломаную линию.

### Требования
1. Класс Point с координатами X, Y
2. Класс Line с начальной и конечной точками
3. Класс Polyline с набором точек
4. Обработка стримом: удаление дубликатов, сортировка, преобразование

### Алгоритм решения
1. Создание классов: Point, Line, Polyline
2. Метод обработки стримом:
    - Удаление дубликатов (distinct())
    - Сортировка по координате X
    - Преобразование: отрицательные Y сделать положительными
    - Сбор в список
3. Создание ломаной из обработанных точек

## Задание 1: Класс Дробь с кэшированием

### Формулировка задачи
Необходимо создать класс Дробь, добавить интерфейс на три метода: получение вещественного значения, установка числителя и установка знаменателя. Сгенерировать такую версию дроби, которая будет кэшировать вычисление вещественного значения.

### Требования к реализации
- Класс `Fraction` с целочисленными полями: числитель и знаменатель
- Дробь создается с указанием числителя и знаменателя
- Может вернуть строковое представление вида "числитель/знаменатель"
- Корректная обработка отрицательных значений: знаменатель не может быть отрицательным
- Переопределение метода сравнения объектов по состоянию
- Две дроби считаются одинаковыми, когда у них одинаковые значения числителя и знаменателя
- Отдельный класс `CachedFraction` для реализации кэширования

### Реализация
Созданы два класса:
- `Fraction` - базовая реализация дроби без кэширования
- `CachedFraction` - версия с кэшированием вещественного значения, наследует от `Fraction`

## Задание 3: Обработка списков

### Формулировка задачи
Создать новый список из элементов, которые присутствуют только в одном из двух исходных списков.

### Требования
- Элементы, которые есть в L1, но нет в L2
- Элементы, которые есть в L2, но нет в L1
- Каждый элемент только один раз в результате
- Не включать элементы, присутствующие в обоих списках

### Алгоритм решения
1. Создание результирующего списка
2. Первый проход по L1: проверка отсутствия в L2
3. Второй проход по L2: проверка отсутствия в L1
4. Проверка уникальности в результате
5. Возврат результата

## Задание 4: Обработка данных абитуриентов

### Формулировка задачи
Прочитать данные из файла об абитуриентах и найти тех, кто не допущен к экзаменам.

### Требования
- Чтение данных из текстового файла
- Формат: количество абитуриентов, затем строки "Фамилия Имя балл1 балл2"
- Допуск: ≥30 баллов по каждому предмету
- Вывод списка не прошедших

1. Чтение файла с помощью BufferedReader
2. Обработка каждой записи: разделение строки, извлечение данных
3. Проверка условий допуска (оба балла ≥30)
4. Хранение данных в Map: ключ - имя, значение - баллы

## Задание 5: Звонкие согласные в тексте

### Формулировка задачи
Файл содержит текст на русском языке. Напечатать в алфавитном порядке все звонкие согласные буквы, которые входят более чем в одно слово.

### Требования к реализации
- Чтение текста из файла
- Определение звонких согласных: б, в, г, д, ж, з, й, л, м, н, р
- Учет только букв, встречающихся в разных словах (более чем в одном слове)
- Вывод в алфавитном порядке
- Использование Set для хранения данных 

### Алгоритм работы
1. Чтение файла построчно
2. Преобразование текста в нижний регистр и очистка от небуквенных символов
3. Разделение на слова
4. Для каждого слова: определение уникальных звонких согласных
5. Подсчет в скольких словах встречается каждая согласная
6. Фильтрация (только те, что в >1 слове) и сортировка
7. Вывод результата

## Задание 7: Обработка точек стримом

### Формулировка задачи
Дан набор объектов типа Point. Необходимо взять все Point в разных координатах, убрать с одинаковыми X,Y, отсортировать по X, отрицательные Y сделать положительными и собрать это все в ломаную (объект типа Polyline).

### Требования к реализации
1. Класс Point:
    - Координата Х: число
    - Координата Y: число
    - Возвращает текстовое представление вида "{X;Y}"
2. Класс Line (Линия):
    - Координата начала: Точка
    - Координата конца: Точка
    - Возвращает текстовое представление вида "Линия от {X1;Y1} до {X2;Y2}"
3. Класс Polyline (Ломаная):
    - Имеет массив Точек, через которые линия проходит
    - Может быть приведена к строковой форме вида "Линия [Т1,T2,…,TN]"

### Реализация
Созданы классы в пакете `pointss`:
- `Point` - с переопределенными `equals()`, `hashCode()`, `toString()`
- `Line` - для представления отрезка
- `Polyline` - для представления ломаной линии
